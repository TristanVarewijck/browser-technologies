LocalStorage

Local Storage vs Session Storage

local storage en sessionstorage zijn bijna hetzelfde je roept ze ook aan met dezelfde API, het verschil zit hem in dat bij sessionstorage de data opgeslagen wordt totdat de “tab” wordt gesloten. Bij Local storage wordt de data opgeslagen totdat de gebruiker zijn cache schoonmaakt of wanneer de web-app dat doet.

    Voorbeelden van dingen die je opslaat in je localstorage:
  • Voortgang
  • Zoektermen
  • Opgeslagen items
Als je iets opslaat in je localstorage moet je natuurlijk wel onthouden dat wanneer iemand zijn cache verwijderd de localstorage ook wordt geleegd. Als je iemand bijvoorbeeld progressie heeft gemaakt op een spel die localstorage gebruikt dat zal alle progressie weg gaan als de de cache wordt verwijderd. In je localstorage zie je de item staan die je hebt opgeslagen zelfs wanneer je de code hierboven weg zal halen zullen deze items hier nog steeds staan opgeslagen. Inspecteren > application > local Storage Hier zie je het object { key: value } wat je ook weer kan ophalen voor bijvoorbeeld een detail pagina. https://www.apple.com/shop/bag Bevindingen over localStorage toegepast op de website van apple Wanneer je inlogt en je slaat iets op dat wordt dit bewaard in de “bag” tab. Op het moment dat je, je jouw cache verwijderd wordt je automatische uitgelogd. Wanneer je weer inlogt staan je items er wel weer, ik denk dat de items worden opgeslagen in een database onder een user ID, waardoor deze niet worden verwijderd met de cache.

Cookies

Cookies (instellingen)

Cookies werken op een manier hetzelfde als localstorage and sessionstorage verschillen zijn dat cookies worden gesupport door alle browsers, ook moet localstorage handmatig worden verwijderd en kunnen er op cookies instellingen worden gezet, wat betekent dat cookies automatische verwijderd kunnen worden. Daarnaast is localstorage alleen beschikbaar op de client side en zijn cookies (data) beschikbaar op client side en server side wat het nog powerful maakt.

Om een simpele cookie te maken doe je simpelweg; document.cookie = “name=Tristan”; name = key, Tristan = value;

    Bevindingen over Cookies toegepast op de website van apple
  1. Bij apple als je de cookies verwijderd worden de items uit je shopping bag verwijderd.
  2. Tegelijkertijd wordt je ook uitgelogd.
  3. Hoe dit precies in zijn werking gaat weet ik niet maar ik weet wel dat er data zit verwerkt in de cookies om de gebruikservaring beter te maken.

Afbeeldingen

Wat zijn afbeeldingen?

Een afbeelding of een illustratie kan meerdere functies hebben. De bekendste vorm is een visuele toelichting bij een verhaal of een beschrijving. Een afbeelding kan ook enkel een verhaal of bewering laten zien zonder een tekst of beschrijving wat erbij hoort, bijvoorbeeld een achtergrond foto van een stad. Ook kan een afbeelding een aanvulling zijn voor een interface om het meer visueel mooier te maken, zoals banners.

Case Apple

Als de afbeeldingen uitgeschakeld zijn op de browser Chrome en als je dan de website van Apple bezoekt, dan komt het heel kaal over vergeleken met als de afbeeldingen wel aan stonden. Op de home pagina zie je dan als koptekst tekst daan over de iPhone 13 Pro. Onder de teksten is er dan een groot leeg ruimte. Daar hoort dan een afbeelding te staan maar als de afbeeldingen uit staan is het leeg. Er staat ook geen alt tekst bij wat wel heel handig geweest zou zijn als dat er wel was geweest.

Op de voorbeeld hieronder kan de gebruiker zonder een afbeelding te zien weinig begrijpen wat de context is. De gebruiker leest alleen ‘Vol kleur. Vol vermogen’. Wat moet de gebruiker hiervan begrijpen? Zonder een alt tekst is het ook heel onduidelijk. Op de voorbeeld hieronder is het wel goed, een afbeelding met een boodschap moet een alt tekst hebben om uit te leggen wat voor afbeelding het is.

Custom Fonts

Wat zijn Custom Fonts?

Custom Fonts zijn aangepaste, zelfgemaakte fonts om een ander lettertype te weergeven. Een goed voorbeeld hiervan is een icon font. Deze custom font laat in plaats van letters iconen zien. Bijvoorbeeld als de gebruiker de letter ‘A’ typt, komt er een icoontje (bijvoorbeeld een opslaan icoon) tevoorschijn in plaats van de letter ‘A’. Een ander voorbeeld voor custom fonts zijn lettertypes wat uniek gemaakt zijn. Een voorbeeld is de lettertype Zaans. (https://www.zaans.nl/uploads/media/5ec515bbe5798/zaans-brandbook-v2-1-2018.pdf). Dit lettertype is uniek want het is speciaal gemaakt voor de website Zaans.nl waarbij mensen informatie kan lezen over bepaalde activiteiten zoals de Zaanse Schans.

JavaScript

Hoe functioneel is de website van Apple?

  • JavaScript is een veelgebruikte scripttaal om webpagina's interactief te maken en webapplicaties te ontwikkelen. Naast HTML en CSS is JavaScript een van de kerntechnologieën van het wereldwijde web. (bron: https://nl.wikipedia.org/wiki/JavaScript)
  • Hoeveel mensen zetten ongeveer hun JavaScript uit: volgens dit artikel (bron: How Many Users Have JavaScript Disabled - Search Engine ...https://www.searchenginepeople.com › stats-no-javascript) zal het tussen de 0 en 2 procent zijn. Het gemiddelde zal zo’n 1,3% zijn. Andere artikelen (wel van een aantal jaar geleden) leggen het gemiddelde op 0,2/0,1%.
  • Als ik bij Apple de JavaScript uitzet (dit kan in safari bij het tabje ontwikkel en dan helemaal onderin naar “Schakel JavaScript uit”) dan zul je gelijk merken dat verschillende functies verdwijnen. Geen popup om je locatie of taal aan te passen, ook worden sommige afbeeldingen langzamer ingeladen en zullen veel animaties niet meer werken. We kennen allemaal wel de toffe animaties van Apple op hun product pagina’s. Deze werken dan niet echt meer. Deze vertrouwen nog teveel op JS.
  • Ook veel support websites van Apple werken niet meer en geven aan dat je Browser JS nodig heeft om te kunnen draaien en werken.

Kleur

Kleur uitzetten & kleurenblindheid instellen

  • Via https://www.toptal.com/designers/colorfilter/ kun je heel gemakkelijk je website checken met filters die de verschillende soorten kleurenblind tonen. Het is gek om dit zien, de website blijft wel werken. Het enige probleem hier is dat producten natuurlijk anders er uit kunnen zien. Hier kan Apple natuurlijk ook niet heel veel aan doen. Ik heb niet het gevoel dat als je kleurenblind bent je veel problemen zal hebben op de website van Apple.
  • Meer over kleurenblindheid https://en.wikipedia.org/wiki/Color_blindness
  • Goed artikel van CSS Tricks over het testen van kleurenblindheid in je website (in dit artikel staan nog meer goede artikelen, ja wat een inception). https://css-tricks.com/accessibility-basics-testing-your-page-for-color-blindness/
  • Ik had 1 klein foutje gezien, van een licht grijze tekst op een witte achtergrond. Voor de rest voldoet Apple wel aan de WCAG. Hier meer over WCAG https://wcag.nl
  • Apple werkt zonder kleur (grayscale) ook goed. Dit komt ook omdat ze veel zwart, wit en grijs gebruiken in hun basis stijl.
